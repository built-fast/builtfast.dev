### New Features

**`POST /vector/environments/{env}/dns/recheck`**

- Re-verifies a custom domain's DNS configuration on demand. Resolves the customer's CNAME, compares against the expected target (Bunny CDN hostname for production with CDN, otherwise the Ymir vanity domain), and marks the environment as DNS-verified when matched.
- Returns `200` with `"DNS verified"` on success or `"DNS not yet propagating to expected target"` when still pending.
- Returns `422` when the environment has no custom domain to verify.

### API Improvements

**Environment responses**

- Added `dns_status` object with `state`, `records`, and `last_checked_at`. The `records` array lists CNAMEs the customer must publish (each with `type`, `host`, `value`, and `purpose` of `traffic` or `validation`).
- Possible `state` values: `verified`, `pending`, `not_required`.

**Site environment summaries**

- Added `dns_status_state` and `requires_dns_action` to the environment objects embedded in site responses, allowing dashboards to flag environments needing customer DNS updates without a separate fetch.
