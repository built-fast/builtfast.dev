---

The Create Site endpoint now supports automatic WordPress installation during site provisioning.

<!--more-->

### New Feature

- Added optional `wp_admin_email` parameter to trigger WordPress auto-install on the dev container at first boot
- Added optional `wp_admin_user` parameter to customize the WordPress admin username (defaults to "admin", max 60 characters)
- Added optional `wp_site_title` parameter to set the WordPress site title (defaults to "WordPress", max 255 characters)
- When `wp_admin_email` is provided, the creation response includes a `wp_admin` block with the generated admin credentials (`user`, `email`, `password`, `site_title`)
- The `wp_admin` block is **only returned in the creation response** -- store the password at that time

See the [Create Site endpoint documentation](/api/#create-site) for details.
