A new `PATCH /account` endpoint lets partners configure an external customer URL template, plus site termination status labels and WAF parameters have been updated.

<!--more-->

### New Features

**Update Account Settings**

`PATCH /api/v1/vector/account` updates partner-level account configuration. The first supported setting is `external_customer_url`, a template containing the literal token `YOUR_ID` that is substituted with each site's `your_customer_id` when rendered in the control panel, turning customer IDs into links back to your own system.

- Pass `null` to clear the value
- Returns the same shape as `GET /vector/account` so you can refresh cached state in one round trip
- A `partner.external_customer_url` field is now included on `GET /vector/account` responses

### Breaking Changes

**Site termination status renamed**

The `terminating` status on sites has been renamed to `termination_requested`:

| Before              | After                  |
| ------------------- | ---------------------- |
| `terminating`       | `termination_requested` |
| `Terminating`       | `Termination Requested` |

**WAF `disabled_rule_groups` removed**

The `disabled_rule_groups` parameter has been removed from WAF settings endpoints. Use `disabled_rules` to disable individual rules instead.

### Action Required

- Update any integrations checking for the `terminating` site status to match `termination_requested`
- Remove `disabled_rule_groups` from WAF settings payloads

See the [Vector Pro API documentation](/api/) for full details.
