---

The `build_output` field has been removed from deployment responses. Build output is now included inline in `stdout` alongside the rest of the deployment log.

<!--more-->

### Breaking Changes

**`build_output` removed from deployment responses**

Deployment objects returned from the API previously exposed two separate log streams: `stdout` (deployment output) and `build_output` (build-phase output). These have been consolidated -- the build phase is now streamed into `stdout` so a single field contains the complete deployment log.

| Before | After |
| --- | --- |
| `stdout` (deploy log) + `build_output` (build log) | `stdout` (full deployment log, including build output) |

Affected endpoints include:

- `GET /vector/sites/{site}/environments/{environment}/deployments`
- `GET /vector/sites/{site}/environments/{environment}/deployments/{deployment}`

### Action Required

- Remove references to `build_output` from any clients reading deployment responses
- Read build-phase output from `stdout` instead

See the [Vector Pro API documentation](/api/) for full details.
