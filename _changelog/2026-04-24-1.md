date: 2026-04-24 14:35:00 -0500
tags: [vector-pro, breaking]

The `environment` query parameter on `GET /vector/sites/{site}/logs` is now required and must match an environment that belongs to the site.

<!--more-->

### Breaking Changes

**`GET /vector/sites/{site}/logs`**

- `environment` is now a required query parameter. Requests without it will be rejected.
- The parameter must reference an environment (`prod`, `staging`, `dev`, etc.) that belongs to the target site.
- Example values have been updated from `production` to `prod` to match platform naming.

### Action Required

- Update clients that call the logs endpoint to always supply `environment`.
- Replace legacy `production` values with the correct environment name (typically `prod`).

See the [Vector Pro API documentation](/api/) for full details.
