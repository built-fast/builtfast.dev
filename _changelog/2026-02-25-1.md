New Restores API endpoints and a backup response status correction.

<!--more-->

### New Features

- **Restores API** -- new endpoints for managing on-demand site restores from backup snapshots:
  - `GET /api/v1/vector/sites/{site}/restores` -- list restores with pagination
  - `GET /api/v1/vector/sites/{site}/restores/{restore}` -- get restore details
  - `POST /api/v1/vector/sites/{site}/restores` -- create a restore from a completed backup
- Restore options include `scope` (full, database, files), `search_replace` pairs, `drop_tables`, and `disable_foreign_keys`
- Restore status lifecycle: `pending` -> `processing` -> `completed` -> `failed`

### API Improvements

- Create Backup endpoint now returns `202 Accepted` instead of `201 Created` to better reflect the async nature of the operation
