title: "Vector Pro: Domain Change API Consolidation"
date: 2026-03-11 12:00:00 -0500
tags: [breaking, vector-pro]
---

Domain changes are now handled through the Update Environment endpoint instead of a separate endpoint.

<!--more-->

### Breaking Changes

- **Removed `POST /environments/{env}/domain-change` endpoint** -- domain changes are now initiated by updating `custom_domain` via `PUT /environments/{env}`
- **Removed `name` field** from the Update Environment request body

### API Improvements

- The Update Environment endpoint now returns `202` with a `pending_domain_change` object when `custom_domain` changes, and `200` for metadata-only updates
- Domain change validation errors (`environment not active`, `domain change already in progress`) are now returned directly from the Update Environment endpoint
- Updated `custom_domain` field description to note that changes trigger infrastructure updates (CDN, DNS, SSL)

### Action Required

- Replace any calls to `POST /api/v1/vector/environments/{env}/domain-change` with a `PUT /api/v1/vector/environments/{env}` request that includes the `custom_domain` field
- Remove any `name` field from Update Environment request bodies
- Update response handling to check for `202` status and the `pending_domain_change` object in the response
