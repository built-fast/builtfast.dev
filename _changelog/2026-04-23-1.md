`custom_domain` is now optional when creating an environment, and the create flow's async ACM and Bunny CDN provisioning steps are now documented.

<!--more-->

### API Improvements

**`POST /vector/sites/{site}/environments`**

- `custom_domain` is now optional. Environments created without a custom domain are reachable on their auto-generated `platform_domain` (for example `wispy-dust--staging-2.vectorpages.com`).
- When `custom_domain` is supplied, ACM certificate provisioning is initiated asynchronously. For production environments, the hostname is also attached to the Bunny.net pull zone.
- `custom_domain_certificate.dns_validation_records` and the CDN `dns_target` are not available in the create response -- poll `GET /vector/sites/{site}/environments/{environment}` to retrieve them once provisioning progresses.
- A new example response illustrates the shape returned when `custom_domain` is omitted (`custom_domain: null`, `custom_domain_certificate: null`, `dns_target` equal to the platform domain).

See the [Vector Pro API documentation](/api/) for full details.
