What is OpenAPI?

OpenAPI is a standard format for describing REST APIs. It's a YAML or JSON file that documents every endpoint, parameter, request body, and response shape in a structured, machine-readable format.

You can feed the spec to code generators to produce typed SDKs, import it into tools like Postman, or use it as context for AI coding assistants.

The Vector Pro spec follows OpenAPI 3.1 {% include icons/external-link.html size="xs" %}.

Spec Details

Version
OpenAPI 3.1
Format
YAML
Base URL
api.builtfast.com/api/v1/vector
Auth

Content-Type

{% include components/section-header.html label="// usage" title="What you can do with it" %}

Generate typed clients

Use OpenAPI Generator to produce SDKs in TypeScript, Python, Go, PHP, or 50+ other languages.

openapi-generator-cli generate -g typescript-fetch

Import into API tools

Drop the spec into Postman, Insomnia, HTTPie, or VS Code extensions to get a working collection.

Postman: Import → OpenAPI → URL or File

Use with AI assistants

Attach the spec to Claude, ChatGPT, or Cursor. The structured format provides complete API context.

"Using this spec, write a service that..."
// client generation

Generate a client

bash 
# Install the generator
npm install @openapitools/openapi-generator-cli -g

# Generate a TypeScript client
openapi-generator-cli generate \
  -i https://builtfast.dev/api/openapi.yaml \
  -g typescript-fetch \
  -o ./vector-client

# Or Python
openapi-generator-cli generate \
  -i https://builtfast.dev/api/openapi.yaml \
  -g python \
  -o ./vector-client

See all available generators {% include icons/external-link.html size="xs" %}.

// versioning

Staying up to date

The spec is updated when the API changes. For CI pipelines, fetch directly from the URL.

bash 
# Fetch the latest spec
curl -O https://builtfast.dev/api/openapi.yaml

# Or generate directly in CI
openapi-generator-cli generate \
  -i https://builtfast.dev/api/openapi.yaml \
  -g typescript-fetch \
  -o ./src/api-client
