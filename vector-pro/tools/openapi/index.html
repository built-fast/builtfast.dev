What is OpenAPI?

OpenAPI (formerly Swagger) is the industry standard for describing REST APIs. An OpenAPI specification is a machine-readable YAML or JSON file that documents every endpoint, parameter, request body, and response in an API.

With an OpenAPI spec, you can generate API clients in any language, create interactive documentation, validate requests, and integrate with hundreds of developer tools.

Vector Pro OpenAPI Spec

Our OpenAPI 3.1 specification covers the complete Vector Pro API, including all endpoints, authentication requirements, request/response schemas, and error formats.

Common Use Cases

Generate API Clients

Use OpenAPI Generator to create typed clients in your language of choice: 

# Install OpenAPI Generator
npm install @openapitools/openapi-generator-cli -g

# Generate a TypeScript client
openapi-generator-cli generate \
  -i https://builtfast.dev/vector-pro/api/openapi.yaml \
  -g typescript-fetch \
  -o ./vector-pro-client

# Generate a Python client
openapi-generator-cli generate \
  -i https://builtfast.dev/vector-pro/api/openapi.yaml \
  -g python \
  -o ./vector-pro-client

# Generate a Go client
openapi-generator-cli generate \
  -i https://builtfast.dev/vector-pro/api/openapi.yaml \
  -g go \
  -o ./vector-pro-client

Import into API Tools

Most API development tools support OpenAPI import:

  • Postman — Import → OpenAPI to create a collection
  • Insomnia — Import/Export → From URL or file
  • HTTPie — Use with the OpenAPI plugin
  • VS Code — REST Client and Thunder Client extensions

Generate Documentation

Tools like Redocly, Stoplight, and Swagger UI can render beautiful API documentation from the spec.

Pro Tip

Build Integrations with AI Assistants

AI coding assistants like Claude, ChatGPT, and Cursor work exceptionally well with OpenAPI specs. Paste the spec into your conversation and ask for exactly what you need—the structured format gives the AI complete context about endpoints, parameters, and response shapes.

Example prompt:

# Attach openapi.yaml to the conversation, then:

"Using the attached Vector Pro OpenAPI spec, write a Laravel service class that creates a new site, polls until it's active, creates a production environment, and triggers a deployment. Include proper error handling and return a DTO with the site and environment details."

The AI will generate working code that matches the exact API contract—correct endpoints, required parameters, response types, and error handling.

Keeping Up to Date

The OpenAPI spec is versioned alongside the API. When new endpoints or parameters are added, the spec is updated. For automated workflows, fetch the spec directly from the URL rather than storing a local copy: 

# Always get the latest spec
curl -O https://builtfast.dev/vector-pro/api/openapi.yaml
