What is Postman?

Postman is an API platform for building, testing, and documenting APIs. It provides a user-friendly interface for making HTTP requests, organizing them into collections, and sharing them with your team.

If you're new to Postman, you can download the desktop app or use the web version.

Vector Pro Collection

Our official Postman collection includes all Vector Pro API endpoints, pre-configured with variables and example requests. It's the fastest way to explore the API without writing code.

Getting Started

1. Import the Collection

Download the collection file above, then import it into Postman: File → Import (or drag and drop the file). The collection will appear in your sidebar.

2. Configure Environment Variables

The collection uses variables for authentication and common values. Create a new environment in Postman and set:

Variable Description Example
base_url API base URL https://api.builtfast.com
api_token Your API bearer token your-api-token
site_id A site ID for testing 01JFGXK4NQRST5VWX9YZ0ABCDE
environment_id An environment ID for testing 01JFGXK4NQRST5VWX9YZ0ABCDG

3. Make Your First Request

With your environment selected, open the "Sites" folder and try the "List Sites" request. You should see your sites returned in the response.

Collection Structure

The collection is organized by resource:

  • Sites — Create, list, update, delete, suspend, and manage sites
  • Environments — Create and manage deployment environments
  • Deployments — Trigger deployments and rollbacks
  • Domains — Manage custom domains and SSL
  • Backups — Create and restore backups
  • Logs — Retrieve site and environment logs

Each request includes example responses and documentation inline.

Tips

  • Use the Console — Postman's console (View → Show Postman Console) shows the raw HTTP requests and responses, useful for debugging
  • Save Examples — After making a successful request, save it as an example for future reference
  • Collection Runner — Use the collection runner to execute multiple requests in sequence, great for testing workflows
