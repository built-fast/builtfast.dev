Security

Security Policy

We take security seriously. If you've found a vulnerability, we want to hear from you.

Our Commitment

Security isn't an afterthought for us. With decades of experience in web hosting, we understand the responsibility that comes with handling infrastructure and customer data. Every project we release is built with a security-first mindset.

We appreciate the work of security researchers who help keep our projects and users safe. If you discover a security issue in any of our repositories, please report it responsibly so we can address it promptly.

Reporting a Vulnerability

For security vulnerabilities, please do not open a public GitHub issue. Instead, report privately using one of these methods:

GitHub Security Advisories

Use GitHub's private vulnerability reporting on the specific repository. This is the preferred method for repository-specific issues.

Email Directly

For sensitive disclosures or issues spanning multiple projects, email Josh Priddle directly.

josh@builtfast.com

What to Include

When reporting a vulnerability, please include:

  • A description of the vulnerability and its potential impact
  • Steps to reproduce the issue
  • Affected versions or commits, if known
  • Any suggested fixes or mitigations

What to Expect

  • Acknowledgment within 48 hours of your report
  • Regular updates as we investigate and work on a fix
  • Credit in the security advisory and release notes (unless you prefer to remain anonymous)
  • Coordinated disclosure once a fix is available
Thank You

We genuinely appreciate security researchers who take the time to investigate and report vulnerabilities. You'll receive public credit for your discovery and our sincere gratitude for helping make open source software more secure.
